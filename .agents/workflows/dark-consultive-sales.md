# 🧠 Skill: Venta Consultiva + Psicología Oscura

## Filosofía Central

> **"El cliente nunca debe sentir que le están vendiendo. Debe sentir que le están ayudando a tomar la mejor decisión posible... y que el tiempo se agota para hacerlo."**

Este skill fusiona dos disciplinas:
1. **Venta Consultiva** — El vendedor (o bot) actúa como asesor experto. Hace preguntas, diagnostica necesidades, educa al cliente, y recomienda soluciones personalizadas. El cliente confía porque siente que lo entienden.
2. **Psicología Oscura de Ventas** — Gatillos cognitivos invisibles que aceleran la decisión de compra: escasez, urgencia, prueba social, anclaje, aversión a la pérdida, reciprocidad, identidad tribal, FOMO, y más.

**La combinación es letal**: El cliente baja la guardia porque siente asesoría genuina, y los gatillos operan en el subconsciente empujando a la acción.

---

## CUÁNDO USAR ESTE SKILL

Usar cuando el usuario necesite:
- Crear o mejorar una **landing page de alta conversión**
- Diseñar **prompts de bot de ventas** (WhatsApp, chat, etc.)
- Escribir **copy de ventas** para cualquier canal
- Optimizar un **flujo de venta consultiva** existente
- Aplicar **gatillos persuasivos** a cualquier pieza de contenido
- Crear **secuencias de follow-up** para leads fríos/tibios/calientes
- Diseñar **scripts de objeciones** para vendedores o bots

## PARTE 1: FRAMEWORK DE VENTA CONSULTIVA

### 1.1 Los 5 Pilares de la Venta Consultiva

| Pilar | Principio | En Práctica |
|-------|-----------|-------------|
| **Diagnóstico** | Entender ANTES de recomendar | Hacer 2-3 preguntas antes de ofrecer producto |
| **Educación** | Informar para generar autoridad | Compartir datos, comparativas, tips del sector |
| **Personalización** | Adaptar la solución al individuo | "Para tu caso específico, lo ideal sería..." |
| **Transparencia** | Honestidad sobre limitaciones | "Ese producto no te conviene porque..." |
| **Seguimiento** | Acompañar post-compra | Follow-up a las 24h, 48h, 7 días |

### 1.2 El Flujo Consultivo (para bots y humanos)

```
FASE 1: RAPPORT (0-30 seg)
├── Saludo cálido y personalizado
├── Preguntar nombre (si no se tiene)
├── Micro-validación: "Qué bueno que nos escribes"
└── Señal de disponibilidad: "Estoy aquí para ayudarte"

FASE 2: DIAGNÓSTICO (30 seg - 2 min)
├── Pregunta abierta: "¿Qué estás buscando?" / "¿En qué te puedo asesorar?"
├── Pregunta de contexto: "¿Ya has usado [producto/servicio]?"
├── Pregunta de preferencia: "¿Tienes alguna preferencia de [variable]?"
└── GATILLO SECRETO: Cada pregunta genera micro-compromiso (consistency bias)

FASE 3: RECOMENDACIÓN EXPERTA (2-4 min)
├── Resumen del diagnóstico: "Entiendo, entonces buscas X porque Y"
├── Recomendación principal: "Para tu caso, lo ideal es [Producto A]"
├── Alternativa: "También podrías considerar [Producto B] si [condición]"
├── Educación sutil: "La diferencia entre A y B es que..."
└── GATILLO SECRETO: Dar una razón "porque" (justification bias)

FASE 4: CIERRE NATURAL (4-6 min)
├── Check de dudas: "¿Tienes alguna otra pregunta?"
├── Facilitar la acción: "¿Te lo separo?" / "¿Quieres que te arme el pedido?"
├── Reducir fricción: "Es super fácil, solo necesito [X]"
└── GATILLO SECRETO: "¿Te lo envío hoy o mañana?" (cierre alternativo)

FASE 5: POST-VENTA CONSULTIVA
├── Confirmar pedido con resumen
├── Follow-up 24h: "¿Todo bien con tu pedido?"
├── Follow-up 7d: "¿Cómo te fue con [producto]?"
└── GATILLO SECRETO: La atención post-venta genera reciprocidad para recompra
```

### 1.3 Reglas de Oro del Asesor Consultivo

```
HACER:
✅ Escuchar más de lo que hablas (ratio 60/40)
✅ Hacer preguntas abiertas antes de recomendar
✅ Decir "no sé" cuando no sepas — y buscar la respuesta
✅ Recomendar el producto correcto aunque sea el más barato
✅ Dar alternativas (nunca un solo camino)
✅ Usar el nombre del cliente
✅ Validar emociones: "Entiendo tu preocupación"
✅ Educar con datos concretos, no opiniones vagas

NO HACER:
❌ Empujar el producto más caro sin justificación
❌ Inventar datos o características
❌ Ignorar objeciones — abordarlas de frente
❌ Presionar cuando el cliente dice "lo voy a pensar"
❌ Hacer promesas que no puedes cumplir
❌ Copiar respuestas genéricas — cada cliente es único
❌ Hablar mal de la competencia directamente
```

### 2.2 Documentación de Gatillos en HTML

Cada gatillo implementado en una landing page DEBE documentarse con un comentario HTML:

```html
<!-- GATILLO: [Nombre del Gatillo] — [Sesgo cognitivo que explota] -->
<div class="urgency-badge">🔴 Solo 4 disponibles hoy</div>
```

Esto permite:
- Auditar qué gatillos están activos en cada página
- Optimizar/rotar gatillos basándose en resultados
- Enseñar al equipo cómo funciona cada elemento
- Detectar redundancia o saturación de gatillos

### 4.2 Template de Copy por Sección

#### Hero — Headline (Dolor→Agitación→Solución)

**Fórmula**: [Problema del cliente] + [Consecuencia de no actuar] + [Tu solución]

```
MALO:  "Los mejores productos del mercado"  ← genérico, no hay dolor
BUENO: "Tu Salud No Puede Esperar Más"      ← nombra el riesgo de no actuar
MEJOR: "Deja de [Problema]. [Solución] en [Tiempo/Facilidad]"
```

**Ejemplos por nicho:**
- Médico: "Tu Salud No Puede Esperar Más"
- Fitness: "Deja de Soñar el Cuerpo que Quieres. Constrúyelo."
- Vapes: "¿Cansado de Productos Mediocres? Mereces Calidad Real."
- Ropa: "Tu Estilo Habla Antes Que Tú. Haz Que Diga Lo Correcto."

```javascript
function updateCountdown() {
  const now = new Date();

  const h = Math.floor((diff % 86400000) / 3600000);

}
```

## PARTE 11: SISTEMA DE ARQUETIPOS DE CLIENTE

### 11.1 Concepto: ¿Por Qué Arquetipos?

> **"Un visitante que siente que la página le habla a ÉL convierte 3-5x más que uno que ve una página genérica."**

Los arquetipos de cliente segmentan tu audiencia en 3-6 perfiles de compra basados en **dolor + objetivo + nivel de experiencia**. Cada arquetipo tiene:
- Un **perfil psicológico** (dolor, miedo, motivación)
- Un **stack de productos** mapeado al catálogo real
- Un **copy personalizado** (headline, subtexto, CTA)
- Un **color y emoji** que lo identifica visualmente
- Un **testimonial** relevante a su situación
- Un **mensaje de WhatsApp** pre-armado con su objetivo

### 11.6 Mini-Quiz — Lógica de Auto-Selección

```javascript
let quizData = {};

function quizAnswer(step, value) {
  if (step === 1) {
    quizData.experience = value;
    // Mostrar paso 2
    document.getElementById('quizStep1').style.display = 'none';
    document.getElementById('quizStep2').style.display = 'block';
  }
  if (step === 2) {
    quizData.goal = value;
    // Mapear experiencia + objetivo → arquetipo
    let archetype = resolveArchetype(quizData.experience, quizData.goal);
    selectArchetype(archetype);
  }
}

// Tabla de resolución configurable por nicho
function resolveArchetype(experience, goal) {
  const MATRIX = {
    nuevo:       { masa: 'principiante', definir: 'principiante', rendimiento: 'principiante', salud: 'wellness' },
    intermedio:  { masa: 'masa', definir: 'quemar', rendimiento: 'avanzado', salud: 'wellness' },
    avanzado:    { masa: 'masa', definir: 'quemar', rendimiento: 'avanzado', salud: 'wellness' }
  };
  return MATRIX[experience]?.[goal] || 'principiante';
}
```

## PARTE 12: MICRO-INTERACCIONES — "VIDEOGAME FEEL"

### 12.1 El Problema: Páginas Muertas

> **"Si tu página se ve igual que una foto, perdiste. El usuario necesita sentir que la página RESPIRA, que reacciona a él, que lo guía. Como un buen videojuego: tutorial invisible."**

Una landing page con contenido perfecto pero sin movimiento pierde hasta un 40% de interacción. Los elementos interactivos (botones, cards, selectores) deben **comunicar que son tocables** sin necesidad de texto explicativo.

### 12.2 Los 7 Principios del "Videogame Feel"

| # | Principio | Qué resuelve | Implementación |
|---|-----------|-------------|----------------|
| 1 | **Entrada escalonada** | El usuario nota cada elemento | Cards aparecen una a una con bounce |
| 2 | **Respiración** | Muestra que algo está VIVO y es interactivo | Borders que pulsan suavemente |
| 3 | **Instrucción contextual** | El usuario sabe QUÉ HACER | Prompt con emoji animado: "👆 Elige tu objetivo" |
| 4 | **Preview al hover** | El usuario ve el resultado antes de actuar | Texto reactivo que cambia al pasar sobre cada opción |
| 5 | **Feedback satisfactorio** | El click se siente BIEN, como presionar un botón de Xbox | Bounce + checkmark + glow + dimming de no-seleccionados |
| 6 | **Guía de scroll** | El usuario sabe que hay más contenido | Flechas animadas "Desliza para explorar" |
| 7 | **Transición orgánica** | Las secciones no aparecen de golpe | Fade-up, slide-in, scale al entrar en viewport |

### 12.3 Entrada Escalonada (Staggered Entrance)

Cada card/elemento interactivo aparece con un delay de 150ms respecto al anterior. Crea efecto de "menú de videojuego cargando".

```css
/* Cada card empieza invisible y aparece con bounce */
.interactive-card {
  opacity: 0;
  transform: translateY(30px) scale(0.9);
  animation: cardEntrance 0.6s cubic-bezier(0.34, 1.56, 0.64, 1) forwards;
}

/* Delay escalonado — 150ms entre cada una */
.interactive-card:nth-child(1) { animation-delay: 0.8s; }
.interactive-card:nth-child(2) { animation-delay: 0.95s; }
.interactive-card:nth-child(3) { animation-delay: 1.1s; }
.interactive-card:nth-child(4) { animation-delay: 1.25s; }
.interactive-card:nth-child(5) { animation-delay: 1.4s; }

@keyframes cardEntrance {
  0%   { opacity: 0; transform: translateY(30px) scale(0.9); }
  60%  { opacity: 1; transform: translateY(-5px) scale(1.03); } /* Overshoot = satisfactorio */
  100% { opacity: 1; transform: translateY(0) scale(1); }
}
```

**Por qué funciona:** El overshoot (sobrepasa y vuelve) activa el mismo placer del rebote de una pelota. El delay escalonado crea ritmo visual.

### 12.4 Breathing Glow (Respiración)

Los bordes de las cards pulsan suavemente para indicar que son interactivas SIN ser agresivo.

```css
/* Pulso sutil de borders — ritmo de "respiración" */
.interactive-card:not(.active) {
  animation: cardEntrance 0.6s forwards, cardBreathe 3s ease-in-out infinite;
}

/* Cada card respira en un momento distinto (evita sincronía robótica) */
.interactive-card:nth-child(1):not(.active) { animation-delay: 0.8s, 1.5s; }
.interactive-card:nth-child(2):not(.active) { animation-delay: 0.95s, 2.0s; }
.interactive-card:nth-child(3):not(.active) { animation-delay: 1.1s, 2.5s; }

@keyframes cardBreathe {
  0%, 100% { border-color: var(--border); }
  50%      { border-color: var(--border-light); }
}
```

**Opcional — Outer glow en hover:**
```css
.interactive-card::after {
  content: '';
  position: absolute;
  inset: -2px;
  border-radius: inherit;
  box-shadow: 0 0 20px rgba(var(--card-color-rgb), 0.25);
  opacity: 0;
  transition: opacity 0.4s;
  z-index: -1;
}
.interactive-card:hover::after { opacity: 1; }
```

### 12.6 Preview al Hover (Pre-visualización)

Texto reactivo que cambia al pasar sobre cada opción. El usuario "ve" qué va a pasar antes de actuar.

```html
<div class="hover-preview" id="hoverPreview"></div>
```

```javascript
const PREVIEWS = {
  opcion1: '🌱 "Te armamos un combo simple y efectivo"',
  opcion2: '💪 "Proteína + calorías — el stack completo"',
  opcion3: '⚡ "Calidad premium. Para atletas serios"',
};

cards.forEach(card => {
  card.addEventListener('mouseenter', () => {
    const preview = document.getElementById('hoverPreview');
    preview.textContent = PREVIEWS[card.dataset.key] || '';
    preview.classList.add('visible');
  });
  card.addEventListener('mouseleave', () => {
    preview.classList.remove('visible');
  });
  // Mobile equivalent
  card.addEventListener('touchstart', () => {
    preview.textContent = PREVIEWS[card.dataset.key] || '';
    preview.classList.add('visible');
  }, { passive: true });
});
```

```css
.hover-preview {
  min-height: 24px;
  text-align: center;
  font-size: 0.85rem;
  color: var(--text-muted);
  font-style: italic;
  opacity: 0;
  transition: all 0.3s ease;
}
.hover-preview.visible { opacity: 1; }
```

### 12.10 Errores Comunes a Evitar

| ❌ Error | ✅ Correcto |
|---------|-----------|
| Que TODO se mueva a la vez — Satura | Escalonar: cada elemento 150ms después del anterior |
| Animaciones largas (>1s) — Impaciencia | 0.3-0.6s para entradas, 0.2-0.4s para feedback |
| Hover effects solo en desktop | Agregar `touchstart` como equivalente de `mouseenter` |
| Prompt permanente — Ruido visual | Desaparece al primer click del usuario |
| Cubic-bezier lineal — Se siente robótico | Usar `(0.34, 1.56, 0.64, 1)` para bounce natural |
| Olvidar reduced-motion — Accesibilidad | `@media (prefers-reduced-motion: reduce) { animation: none }` |
